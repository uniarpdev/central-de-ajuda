# GEMINI.md - Central de Ajuda Uniarp

## Project Overview
This project is the **Central de Ajuda Uniarp** (Uniarp Help Center), a documentation site built with [Docusaurus 3](https://docusaurus.io/). It serves as a guide for users on how to use Uniarp's systems.

- **Primary Technologies:** React, TypeScript, MDX, Docusaurus.
- **Language:** Portuguese (pt-BR).
- **Deployment:** GitHub Pages (hosted at `https://uniarpdev.github.io/central-de-ajuda/`).

## Building and Running

The project uses `npm` (though the README mentions `yarn`, `package.json` and the environment suggests standard npm/node usage).

### Development Server
Starts a local development server with live reload:
```bash
npm start
```

### Build
Generates static content for production in the `build/` directory:
```bash
npm run build
```

### Type Check
Runs TypeScript type checking:
```bash
npm run typecheck
```

### Deployment
Deploys the site to GitHub Pages:
```bash
npm run deploy
```

## Project Structure
- `docs/`: Markdown/MDX files for the documentation.
- `blog/`: Markdown/MDX files for blog posts.
- `src/`: React source code.
  - `components/`: Custom React components.
  - `css/`: Global and custom CSS styles.
  - `pages/`: Individual site pages (e.g., landing page).
- `static/`: Static assets like images, icons, and files.
- `docusaurus.config.ts`: Main configuration file for the site (metadata, themes, plugins).
- `sidebars.ts`: Configuration for the documentation sidebar (currently set to auto-generate based on file structure).

## Development Conventions

- **Documentation:** All documentation should be placed in the `docs/` directory using Markdown (`.md`) or MDX (`.mdx`).
- **Styling:** Use CSS Modules (e.g., `index.module.css`) for component-specific styles and `src/css/custom.css` for global overrides.
- **Language:** Content should be written in **Portuguese (pt-BR)**.
- **Components:** New components should be written in TypeScript (`.tsx`) and placed in `src/components/`.
- **Images:** Place images in `static/img/` and reference them using absolute paths (e.g., `/img/logo.svg`).

## Common Tasks

- **Adding a new document:** Create a `.md` or `.mdx` file in `docs/`. It will be automatically added to the sidebar.
- **Modifying the Homepage:** Edit `src/pages/index.tsx` and `src/components/HomepageFeatures/`.
- **Updating Site Metadata:** Edit `docusaurus.config.ts` (title, tagline, navbar, etc.).
- **Updating Sidebar:** The sidebar is autogenerated from `docs/`, but can be manually configured in `sidebars.ts`.
